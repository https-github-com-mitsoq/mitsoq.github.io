In order to easily access different environments we can add aliases for kubectl and stern.
ALIAS         SUPCTL ENV    CLUSTER       NAMESPACE
kb            dev           minikube      default
kubesand      staging       hive          default
kubeprep      pre-prod      keg           pre-prod
kubekeg       keg           keg           default
kubeprod      production    tavern        default
kubeci        ci            taproom       ci
kubeds        ds            taproom       datascience
kubetap       tap           taproom       default
kubeloadtest  loadtest      loadtest      loadtest
